## Problem

The VK-specific services violated clean layered architecture boundaries:
1. `VkFriendsExportService` contained redundant proxy repository methods, hardcoded file path configurations, and a bloated ~150 line export job runner.
2. `VkGroupsService` violated the Dependency Rule by depending directly on SQLAlchemy's `AsyncSession` and DB models (`VkGroup`, `VkPost`, `VkComment`, `VkAuthor`), executing raw SQL queries directly in its service logic.

## Root Cause

Lack of architectural discipline during implementation led to database concerns leaking directly into the application service layer, preventing proper domain decoupling and unit testing.

## Solution

1. **VK Friends Service**:
   - Removed all repository proxy methods (`create_job`, `append_log`, `update_progress`, `complete_job`, `fail_job`, `save_friends_batch`, `get_job_by_id`, `get_job_logs`).
   - Split `run_export_job` into cohesive private helpers: `_fetch_all_friends`, `_save_to_database`, and `_generate_xlsx`.
   - Extracted mapping fields and Russian translation headers to `constants.py`.
   - Moved export directory config to `config.py`.
   - Updated the FastAPI router `app/api/routers/vk_friends.py` to fetch job status and log entries via `VkFriendsRepository` instead of the service.
2. **VK Groups Service**:
   - Created `soft_delete_group` abstract method in `IngestionRepository` domain interface.
   - Implemented `soft_delete_group` and mapped `get_active_group_ids` in `SqlAlchemyIngestionRepository`, moving all raw queries and database model deletions there.
   - Removed SQLAlchemy session dependency from `VkGroupsService`.
3. **Tests & Startup**:
   - Modified all test suites to invoke repository layer methods for job updates and status changes.
   - Fixed all 48 tests.

## Prevention

- Keep Application Services decoupled from infrastructure dependencies (e.g. SQLAlchemy sessions, ORM models, HTTP libraries).
- Interact with databases solely through abstract Domain Repository interfaces.
- Enforce strict separation: Routers call Repositories for CRUD, and call Services only for orchestrating complex business processes (like exports or integrations).
