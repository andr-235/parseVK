# VK parsing tasks remain running indefinitely

**Severity:** high

## Problem

VK parsing ran synchronously in the Kafka consumer transaction. Two long-running
tasks could occupy the consumer indefinitely, and a crashed process left both the
local run and the public task in `running` for days. No task-wide timeout, network
timeout, heartbeat, or recovery owner existed.

The ingestion pipeline also assigned completion fields to a frozen `VkTaskRun`
domain dataclass. This raised `FrozenInstanceError` on both the success path and
the exception handler, while the detached mutations could never persist to the
database even if the entity were mutable.

## Root Cause

Kafka delivery and expensive ingestion were treated as one lifecycle. The design
assumed a consumer process would stay alive until every VK request completed and
that mutating a detached domain entity would update its ORM row. Tests used mutable
`SimpleNamespace` fixtures, so the frozen-entity regression was not exercised.

## Solution

Kafka handling now durably enqueues a run and commits quickly. A bounded worker
pool claims rows with `FOR UPDATE SKIP LOCKED`, records a fenced lease owner,
renews a heartbeat, checkpoints ingestion per post, and applies network and
whole-task timeouts. Expired leases are reclaimed with bounded attempts; terminal
state is persisted through repository updates instead of entity mutation.

The migration adds claimable lease fields and requeues legacy `running` rows.
Cancel and resume transitions lock the public task row and use run-scoped outbox
dedupe keys so later recovery cycles can emit fresh events safely. Worker logs
include stable task, run, attempt, and worker identifiers for production checks.

## Prevention

- Keep message acknowledgement separate from long-running work.
- Require leases, heartbeats, bounded concurrency, and retry ceilings for durable workers.
- Persist lifecycle changes through repositories; never mutate frozen domain snapshots.
- Test worker restart, lease loss, timeout, duplicate delivery, and the real domain dataclass.
- Review every external API client for explicit connect/read timeouts.
