# Keywords Form Design

**Дата:** 2026-03-16

## Цель

Исправить форму добавления ключевых слов на фронте так, чтобы:
- кнопка `Добавить` явно работала только при непустом слове;
- поле `Категория` поддерживало комбинированный сценарий: выбор существующей категории из списка и ввод новой вручную.

## Подход

Используем нативный `input + datalist`, потому что этот паттерн уже есть в проекте и он закрывает оба сценария без нового сложного UI. Источник подсказок — уникальные непустые категории из уже загруженного списка `keywords`.

## Изменения

- В `useKeywordsViewModel` добавить вычисление `categorySuggestions`.
- В `KeywordsForm` передавать и отображать `categorySuggestions` через `datalist`.
- Для кнопок `Добавить` и `Добавить фразу` ввести явный `disabled`, основанный на `trim()` соответствующего значения.
- Добавить тесты на состояние кнопки и рендер списка категорий.

## Риски

- `datalist` не дает полностью кастомный UX, но для текущей задачи этого достаточно.
- Нужно сохранить существующее поведение bulk-ввода: после успешного добавления категория не очищается.
