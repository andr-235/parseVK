# Дизайн: категории ключевых слов и теги категорий в комментариях

Дата: 2026-03-16

## Контекст

В проекте уже есть словарь ключевых слов с полем `category`, а в комментариях уже используется логика совпадений по ключевым словам. Пользователь хочет переработать UI так, чтобы:

- список слов на странице словаря был разбит по категориям;
- в комментариях отображались теги категорий;
- по категориям можно было фильтровать и сортировать комментарии.

Важно сохранить текущую модель данных без избыточной миграции и не ломать существующий сценарий работы со словарём и комментариями.

## Цели

- Сделать категории заметной частью UI словаря ключевых слов.
- Показывать категории на карточках комментариев как отдельные теги.
- Дать быстрый фильтр комментариев по одной или нескольким категориям.
- Сохранить поддержку комментариев, у которых совпадения относятся к нескольким категориям.
- Реализовать поведение на основе уже существующего `Keyword.category`.

## Не входит в текущий объём

- Введение отдельной сущности категорий в БД.
- Нормализация или миграция исторических значений категорий.
- Серверная фильтрация комментариев по категориям.
- Персистентное сохранение свёрнутых секций и выбранных фильтров.
- Новая аналитика или агрегаты по категориям сверх UI-фильтрации.

## Подход

Рекомендованный вариант: использовать текущую модель `Keyword.category` как единственный источник истины и построить поверх неё UI-группировку словаря и клиентские теги/фильтры комментариев.

Почему выбран этот вариант:

- не требует изменения схемы данных;
- повторно использует уже существующие keyword matches;
- даёт нужный UX при минимальном риске регрессий;
- оставляет пространство для дальнейшей серверной фильтрации, если она позже понадобится.

## Архитектура

### Страница словаря ключевых слов

Страница словаря сохраняет табличный формат, но рендерит данные секциями по категориям.

Каждая секция содержит:

- название категории;
- количество слов в категории;
- кнопку свернуть/развернуть;
- таблицу строк внутри категории.

Слова без категории попадают в отдельную секцию `Без категории`.

Поиск по словарю продолжает работать по всему набору слов, но результаты показываются внутри соответствующих категорий.

### Страница комментариев

На странице комментариев добавляются два уровня работы с категориями:

- панель фильтров по категориям над списком;
- теги категорий на каждой карточке комментария.

Категории комментария вычисляются из `matchedKeywords`:

- берутся все уникальные непустые `category`;
- один комментарий может иметь несколько тегов;
- если у комментария нет категорий, блок тегов не показывается.

Клик по тегу на карточке синхронизируется с верхним фильтром и включает или выключает категорию в текущем наборе фильтрации.

## Пользовательский сценарий

### Словарь

Пользователь открывает страницу ключевых слов и сразу видит секции категорий. Он может:

- раскрывать только нужные категории;
- быстро оценивать размер каждой категории по счётчику;
- искать слова без потери группировки;
- работать с таблицей в привычном формате внутри секции.

### Комментарии

Пользователь открывает страницу комментариев и видит панель категорий. Он может:

- выбрать одну или несколько категорий;
- увидеть на карточке комментария, к каким категориям он относится;
- кликнуть по тегу у комментария и быстро сузить выдачу;
- просматривать комментарии, которые одновременно относятся к нескольким категориям.

## UI/UX

### Группировка словаря

Базовый layout страницы словаря не меняется: основной паттерн остаётся табличным. Изменяется только представление набора данных:

- категории становятся визуальными группами;
- внутри группы сохраняется текущая таблица;
- по умолчанию секции раскрыты;
- свёрнутое состояние хранится локально в состоянии страницы.

### Теги и фильтры категорий в комментариях

В верхней части страницы комментариев появляется панель фильтрации с мультивыбором категорий.

На карточке комментария:

- теги категорий отображаются как компактные badge-элементы;
- показываются все уникальные категории комментария;
- клик по тегу изменяет глобальный фильтр;
- комментарий остаётся видимым, если совпадает хотя бы с одной выбранной категорией.

Комментарии без категорий:

- отображаются при пустом фильтре;
- скрываются, если выбран хотя бы один фильтр категории.

## Поток данных

### Источник категорий

Источник истины для категорий:

- `Keyword.category` для словаря;
- `matchedKeywords[].category` для комментариев.

Дополнительный серверный слой для категорий не требуется.

### Вычисление категорий комментария

Для каждого комментария вычисляется производное поле вида `commentCategories`:

- уникальные значения `category` из совпавших ключевых слов;
- пустые и `null` значения исключаются;
- порядок можно сделать стабильным, например по алфавиту или порядку первого появления.

### Фильтрация и сортировка

Фильтрация выполняется на клиенте по уже загруженному набору комментариев.

Правила:

- если фильтр пустой, показываются все комментарии;
- если фильтр содержит категории, остаются комментарии, у которых есть хотя бы одно пересечение;
- комментарий с несколькими категориями входит во все подходящие фильтры.

Сортировка в текущем объёме не должна ломать существующий порядок ленты. Рекомендованный режим:

- сначала применяется фильтр;
- затем сохраняется текущая сортировка комментариев;
- при необходимости можно добавить мягкий приоритет комментариев с выбранными категориями, но без полной перегруппировки ленты.

## Обработка сложных случаев

- Если слово не имеет категории, оно показывается в секции `Без категории`.
- Если все слова в результате поиска без категории, страница словаря всё равно остаётся структурированной за счёт секции `Без категории`.
- Если у комментария совпали слова из нескольких категорий, все категории показываются тегами.
- Если совпали только слова без категории, комментарий рендерится без тегов категории.
- Если пользователь включает несколько фильтров, используется логика `OR`, а не `AND`.

## Точки изменения в коде

### Frontend

Ожидаемые изменения сосредоточены в существующих модулях:

- `front/src/modules/keywords` для группировки словаря по категориям;
- `front/src/modules/comments` для вычисления и отображения тегов категорий;
- `front/src/modules/comments/hooks` или `utils` для фильтрации комментариев по категориям;
- общие типы, если потребуется явно описать производные категории комментария.

### Backend

Изменения backend не обязательны, если текущий API уже возвращает у совпавших ключевых слов поле `category`.

Если на одном из путей это поле не приходит стабильно, допустимо ограниченное обновление DTO/mapper без изменения бизнес-логики.

## Тестирование

Нужны проверки трёх уровней.

### Unit

- группировка ключевых слов по категориям;
- выделение уникальных категорий комментария;
- фильтрация комментариев по одной категории;
- фильтрация комментариев по нескольким категориям;
- корректная обработка `null` и пустых категорий.

### Component

- рендер секций категорий на странице словаря;
- сворачивание и разворачивание секции;
- отображение тегов на карточке комментария;
- изменение фильтра по клику на тег.

### Regression

- комментарии без категорий продолжают корректно отображаться;
- слова без категории не пропадают из словаря;
- текущая логика отображения matched keywords не ломается;
- существующая сортировка ленты сохраняется, если фильтр категорий не выбран.

## Риски

- Некачественно заполненные категории в словаре сразу станут заметны в UI.
- Если одна и та же категория записана с разным регистром или опечатками, группировка создаст дублирующиеся секции.
- При полностью клиентской фильтрации результат ограничен текущим загруженным набором комментариев, а не всей базой.

## Критерии готовности

- Страница ключевых слов показывает словарь секциями по категориям.
- Секция `Без категории` отображается для слов без `category`.
- На карточках комментариев видны все уникальные категории, связанные с совпавшими словами.
- Фильтр по категориям работает с мультивыбором.
- Клик по тегу категории на карточке переключает соответствующий фильтр.
- Комментарий с несколькими категориями попадает во все подходящие фильтры.
