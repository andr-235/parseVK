# Безопасность ParseVK

## ⚠️ Критически важно

НИКОГДА не коммитьте секреты в git! После попадания в историю git их невозможно удалить полностью.

## 🔐 Обязательные переменные окружения

Следующие переменные ДОЛЖНЫ быть установлены через `.env` файл:

| Переменная | Описание | Риск при утечке |
|------------|----------|-----------------|
| `VK_TOKEN` | Токен ВК API | Полный доступ к аккаунту ВК |
| `POSTGRES_PASSWORD` | Пароль БД PostgreSQL | Доступ к всем данным |
| `REDIS_PASSWORD` | Пароль Redis | Доступ к кэшу и сессиям |
| `GRAFANA_ADMIN_PASSWORD` | Пароль Grafana | Доступ к мониторингу |

## 🚀 Быстрый старт (безопасно)

```bash
# 1. Скопируйте шаблон
cp .env.production.example .env

# 2. Отредактируйте .env (nano, vim, или редактор)
nano .env

# 3. Заполните ВСЕ пустые поля!
# 4. Убедитесь что .env в .gitignore
grep "^\.env" .gitignore

# 5. Запустите
docker-compose up -d
```

## 🛡️ Что изменено в безопасной версии

### До (опасно):
```yaml
# Порты открыты наружу
ports:
  - "5433:5432"

# Дефолтные пароли в коде
environment:
  POSTGRES_PASSWORD: postgres
  VK_TOKEN: vk1.a.MgXnY9V9Mbxs...  # УТЕЧКА!
```

### После (безопасно):
```yaml
# Порты закрыты (только локально при необходимости)
# ports:
#   - "127.0.0.1:5433:5432"

# Только из переменных окружения
environment:
  POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
  VK_TOKEN: ${VK_TOKEN}
```

## 🔒 Сетевая изоляция

Добавлены отдельные Docker-сети:
- `backend` — БД, Redis, API
- `frontend` — API, Frontend  
- `monitoring` — Prometheus, Grafana, Node-exporter

## 🚨 Чек-лист перед деплоем

- [ ] Все пароли изменены с дефолтных
- [ ] VK токен свежий и действительный
- [ ] Порты БД и Redis закрыты (`# ports:` или `127.0.0.1`)
- [ ] `.env` не коммитится (`git status` не показывает его)
- [ ] Redis защищён паролем (`--requirepass`)
- [ ] Grafana пароль изменён с `admin123`

## 📞 При компрометации

1. **Немедленно** отзовите VK токен: https://vk.com/settings?act=tokens
2. Смените пароли БД и Redis
3. Пересоздайте сервисы: `docker-compose down && docker-compose up -d`
4. Проверьте логи на подозрительную активность
