# Границы микросервисов в parseVK

Проект parseVK представляет собой распределенную систему (монорепозиторий), разделенную на изолированные сервисы. Для предотвращения спагетти-архитектуры ИИ-агенты должны строго уважать следующие границы:

## 1. Слой API Gateway (`api/` или `services/gateway/`)
- Отвечает за маршрутизацию внешних запросов, аутентификацию пользователей и CORS.
- Не содержит прямой бизнес-логики парсинга или работы со сбором данных.
- Общается с другими сервисами через строго описанные HTTP API или брокеры сообщений.

## 2. Слой Сбора Данных / Парсинга VK (`services/vk-parser/` или аналогичный)
- Отвечает за взаимодействие с VK API, обработку rate-limits, парсинг постов, комментариев, авторов.
- Не должен напрямую обращаться к базе данных веб-интерфейса. Работает со своим кэшем/очередями.
- Должен оставаться чисто функциональным сервисом-воркером.

## Правила взаимодействия
- **Запрещено** импортировать внутренние модули одного сервиса в другой напрямую. Используйте общие библиотеки (`libs/`) для кросс-сервисного кода (например, схемы данных, логирование).
- Всегда проверяйте границы при изменении файлов: если задача поставлена для `vk-parser`, файлы gateway не должны изменяться.
