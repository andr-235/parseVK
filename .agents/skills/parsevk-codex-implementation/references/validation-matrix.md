# Матрица валидации parseVK

Этот документ содержит перечень команд проверок (тесты, линтеры, тайпчеки), которые необходимо запускать в зависимости от области вносимых изменений.

| Область изменений | Каталог | Команды проверки |
| :--- | :--- | :--- |
| **Go CLI (`parsevkctl`)** | `tools/parsevkctl-go/` | `go test ./...`<br>`go vet ./...`<br>`go fmt ./...` |
| **Python Backend / Микросервисы** | `services/` | `pytest`<br>`ruff check .`<br>`mypy .` |
| **Фронтенд** | `front/` | `npm test`<br>`npm run lint`<br>`npm run typecheck`<br>`npm run build` |
| **Общие библиотеки (Python)** | `libs/` | `pytest libs/` |

## Правила запуска
1. **Локальность проверок**: Запускайте проверки именно в том каталоге, где были внесены изменения.
2. **Форматирование логов**: При падении тестов извлекайте и анализируйте только первые 3-5 строк ошибок, скрывая остальной вывод, чтобы не перегружать контекст.
3. **Честность**: В хэндоффе пишите точные результаты запуска команд. Если команда не была запущена или завершилась с ошибкой, явно укажите причину.
