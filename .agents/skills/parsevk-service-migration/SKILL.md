# parseVK Service Migration Skill

## Purpose
Навык управляет процессами планирования и реализации миграции микросервисов на новые стандарты (например, перенос legacy-кода на FastAPI).

## When to use
- При планировании рефакторинга или переписывания существующего сервиса.
- При выполнении шагов по миграции кода на FastAPI.
- Для проверки соответствия мигрированного сервиса общим архитектурным требованиям репозитория.

## When not to use
- При исправлении мелких багов в стабильных сервисах.
- При обычном код-ревью в чате.

## Inputs
- Спецификация мигрируемого сервиса (legacy-код, API контракты).
- Архитектурный стандарт FastAPI репозитория.

## Procedure
1. **Анализ существующего сервиса**: Изучите текущие роуты, зависимости, контракты данных и настройки логирования.
2. **Проектирование API-контракта**: Убедитесь, что все существующие API-методы сохранят свои контракты (схемы запросов/ответов, статус-коды) во избежание сбоев во взаимодействии.
3. **Разработка согласно стандартам**: Используйте правила из `references/fastapi-service-standard.md` при проектировании структуры папок нового сервиса.
4. **Поэтапный перенос логики**: Переносите код небольшими изолированными шагами, покрывая новые эндпоинты юнит-тестами.
5. **Валидация**: Выполните полный цикл проверок согласно `references/migration-rules.md`.

## Output format
Отчет о статусе миграции сервиса с перечнем перенесенных эндпоинтов и результатами интеграционных тестов.

## Safety rules
- Запрещено менять контракты публичных API без согласования с архитектором.
- Запрещено удалять legacy-сервис до полного прохождения всех тестов нового сервиса.
