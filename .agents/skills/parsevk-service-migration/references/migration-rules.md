# Правила миграции микросервисов в parseVK

Этот документ содержит регламент выполнения переноса / переписывания микросервисов.

## 1. Подготовка и Исследование
- Соберите список всех внешних вызовов, которые делает сервис.
- Зафиксируйте схему БД и используемые переменные окружения.
- Оцените покрытие тестами. Если покрытия нет, напишите базовые интеграционные тесты перед началом миграции.

## 2. Разработка Новой Версии
- Создавайте новый сервис в изолированной папке под `services/`.
- Следуйте единому FastAPI стандарту репозитория.
- Настраивайте CI/CD джобы для параллельного запуска нового сервиса в тестовом окружении.

## 3. Проверка Контрактов (Contract Testing)
- Схемы запросов и ответов должны быть валидированы с помощью Pydantic.
- Запустите сквозные тесты (End-to-End) взаимодействия нового сервиса с API Gateway.

## 4. Переключение Трафика (Rollout)
- Переключайте трафик постепенно (например, через роутинг в Gateway).
- Мониторьте логи на наличие ошибок 5xx и утечек памяти.
- Окончательно удаляйте старый код только после 7 дней стабильной работы нового сервиса в продакшене.
