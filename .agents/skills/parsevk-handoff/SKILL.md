# ParseVK Handoff

Генерирует стандартизированный финальный отчёт по завершённой задаче в репозитории `andr-235/parseVK`. Используется после реализации задачи, ревью кода или мержа PR.

## Когда использовать

- После завершения реализации задачи (code changes ready)
- После code review
- Перед созданием PR через `parsevkctl task pr`
- После мержа PR (итоговое закрытие задачи)

## Формат отчёта

### Шапка

```markdown
# Handoff: [TASK_TITLE]

**Issue:** #[ISSUE_NUMBER]
**Status:** [Done / Review / Merged]
**Branch:** `[branch-name]`
**Выполнил:** AI Agent
```

### 1. Что было сделано

Краткий список изменений:

- [Feature/Bugfix/Refactor] Краткое описание изменения (файл:строка)
- [Feature/Bugfix/Refactor] Краткое описание изменения

### 2. Scope

- **In scope:** что входило в задачу
- **Out of scope:** что было осознанно исключено

### 3. Проверки

| Проверка | Статус |
|----------|--------|
| pytest / go test | ✅ / ❌ / N/A |
| ruff / lint | ✅ / ❌ / N/A |
| bun run build | ✅ / ❌ / N/A |
| bun run test / vitest | ✅ / ❌ / N/A |

### 4. Архитектурные заметки

- Ключевые решения, принятые в ходе реализации
- Отклонённые альтернативы и причины
- Замечания по backward compatibility

### 5. Известные риски и ограничения

- Описание любых известных проблем
- Что делать при ошибках
- Зависимости, которые нужно учитывать

### 6. Что осталось на потом (если применимо)

- [ ] Незавершённые подзадачи
- [ ] Будущие улучшения
- [ ] Что требует ручного тестирования

### 7. Справочная информация

- **Pull Request:** #PR_NUMBER
- **Base branch:** `fastapi-microservices-rewrite`
- **Ветка:** `[branch-name]`
```

## Правила заполнения

1. **Честность:** Никогда не указывай статус проверки `✅`, если физически не запускал команду
2. **Scope:** Строго следуй разделам `Scope` и `Out of Scope` из описания задачи
3. **Pull Request:** Указывай номер PR, если он создан
4. **Default branch:** `fastapi-microservices-rewrite` (не `main`)
5. **Язык:** Отчёт на русском языке

## Пример

```markdown
# Handoff: Экспорт авторов в CSV

**Issue:** #42
**Status:** Review
**Branch:** `feat/42-export-authors-csv`
**Выполнил:** AI Agent

### 1. Что было сделано

- [Feature] Добавлен эндпоинт `GET /api/v1/authors/export` для экспорта в CSV
- [Feature] Добавлен сервисный слой с генерацией CSV через стандартную библиотеку
- [Test] Добавлены тесты на экспорт с пустым списком и с данными

### 2. Scope

- **In scope:** CSV-экспорт авторов
- **Out of scope:** Экспорт в Excel/JSON, экспорт других сущностей

### 3. Проверки

| Проверка | Статус |
|----------|--------|
| pytest (api-gateway) | ✅ |
| ruff check . | ✅ |

### 4. Архитектурные заметки

- CSV-генерация вынесена в отдельный сервисный слой для тестируемости
- Используется `csv.writer` из stdlib — без дополнительных зависимостей

### 5. Известные риски и ограничения

- Не экранируются запятые в полях — ожидаемое поведение для RFC 4180

### 6. Что осталось на потом

- [ ] Добавить streaming для больших объёмов данных

### 7. Справочная информация

- **Pull Request:** #43
- **Base branch:** `fastapi-microservices-rewrite`
- **Ветка:** `feat/42-export-authors-csv`
```
