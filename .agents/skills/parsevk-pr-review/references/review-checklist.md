# Чеклист рецензирования кода (Code Review Checklist)

Применяйте этот чеклист при анализе Pull Request в проекте parseVK.

## 1. Безопасность (Security)
- [ ] Нет захардкоженных токенов VK, паролей к БД, API-ключей.
- [ ] Конфигурация считывается из переменных окружения через класс настроек.
- [ ] Отсутствуют потенциально уязвимые shell-вызовы с несанитизированными параметрами.
- [ ] Пути файлов валидируются на предмет Path Traversal.

## 2. Архитектура и Границы
- [ ] Сервисы взаимодействуют только через сетевые интерфейсы или общие схемы в `libs/`.
- [ ] Логика не дублирует существующий код в репозитории.
- [ ] Использованы корректные типы данных и аннотации типов (type-safety).

## 3. Стиль и Качество
- [ ] Комментарии добавлены только для сложных, нестандартных или неочевидных решений.
- [ ] Форматирование соответствует принятым стандартам (Ruff для Python, Prettier/ESLint для JS, gofmt для Go).
- [ ] Переменные и функции имеют осмысленные, читаемые названия в camelCase, snake_case или PascalCase.

## 4. Валидация
- [ ] В описании PR предоставлены доказательства локального тестирования (вывод тестов).
- [ ] Все новые функции покрыты юнит-тестами.
