# Правила слияния Pull Request в parseVK

Этот документ содержит формальный список требований, невыполнение любого из которых делает мердж невозможным.

## 1. Состояние ветки и PR
- PR должен иметь статус `OPEN`. Слияние закрытых или смерженных PR запрещено.
- PR не должен быть `DRAFT` (черновик). Перед мерджем переведите его в статус `Ready for review`.
- PR должен быть направлен строго в `main` (или `master`, если настроено иначе).

## 2. Связность с трекером
- В описании PR должна быть строка `Closes #XXX` (где `XXX` — номер Issue).
- Имя ветки должно соответствовать шаблону `<type>/<issue-number>-<short-kebab-summary>`.

## 3. Проверки и Сборка
- Локальные тесты и линтеры должны проходить успешно.
- Внешние проверки (CI/CD) на GitHub должны иметь статус `SUCCESS`.
- Отсутствуют конфликты слияния (merge conflicts).

## 4. Конфиденциальность и безопасность
- Никаких временных файлов разработки (`.env.local`, `.venv/`, кэшей) в коммитах.
- Никаких API ключей, паролей, приватных ключей.
